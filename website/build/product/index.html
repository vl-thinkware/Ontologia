Product

One graph. Every view you need.

Schema, taxonomies, tables, history, validation, exports, API — all backed by the same versioned graph. Edit it from any side; ship it through the same endpoint.

Why teams pick Semlify

Built for the people who actually model meaning.

  • Graph-native

    One engine for schema, taxonomy, and instances. SKOS aligned, OWL-exportable, Neo4j underneath. Not three tools stitched together.

  • Versioned

    Every change is a reversible, attributable event. Tag a snapshot. Diff two tags. Revert a mistake — without leaving the editor.

  • API-first

    Stable JSON-LD / SKOS / OWL endpoints with version pinning, language negotiation, and a built-in playground.

Surfaces

Every view of the same graph.

  • Schema canvas

    An ER-style diagram of your T-Box. Drag classes, draw relations, edit attributes inline. SKOS slots are always present, custom attributes are typed and validated.

    Learn more

  • Taxonomies tree

    Per-scheme broader/narrower hierarchies with drag-drop reparenting. Cycle-safe, embedded with the full concept editor on the right.

    Learn more

  • Tables view

    A sortable, filterable datagrid of every concept in a scheme. Bulk select to deprecate or tag. Backlinks and a live count of incoming relations on every row.

    Learn more

  • API + playground

    Five clean endpoints. Tag pinning, language negotiation, format negotiation. Try every call from the in-app playground before you write a line of integration code.

    Learn more

  • Governance & validation

    Live checks on every edit: orphan concepts, domain/range violations, duplicate prefLabels, deprecated-but-still-referenced. Click to jump to the offender.

    Learn more

  • AI helpers

    Synonyms, translations, duplicate detection, class suggestion. All AI suggestions land as ordinary change events — accept, reject, or revert.

    Learn more
